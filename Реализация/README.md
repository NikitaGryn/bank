# Беларусбанк Инвестиции — демонстрационный прототип

Python-приложение по ТЗ на брокерское обслуживание:

- Django REST API хранит демо-счёт, портфель, заявки и операции.
- Flet предоставляет единый адаптивный интерфейс для web и Android.
- MOEX ISS используется для получения доступных котировок Московской биржи.
- При недоступности API интерфейс продолжает работать на демонстрационных данных.

## Возможности

- демонстрационный вход;
- портфель и брокерский счёт;
- каталог ценных бумаг и поиск;
- карточка инструмента, график и биржевой стакан;
- имитация покупки и продажи с проверкой средств и позиций;
- пополнение и вывод демо-средств;
- заявки и история операций;
- календарь инвестора, калькулятор доходности и поддержка;
- профиль, документы и сведения о счёте депо.

Все операции выполняются только с демонстрационными данными.

## Первый запуск

```powershell
cd C:\Users\user\Desktop\bank\Реализация
py -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

В первом терминале:

```powershell
.\run_backend.ps1
```

Во втором терминале:

```powershell
.\run_web.ps1
```

Или запустить оба компонента:

```powershell
.\start_demo.ps1
```

Web-интерфейс по умолчанию открывается на `http://127.0.0.1:8550`.
Django API работает на `http://127.0.0.1:8000/api`.

## Проверка

```powershell
.\test_all.ps1
```

## Сборка web

```powershell
.\build_web.ps1
```

Результат сохраняется в `dist\web`.

## Сборка APK

```powershell
.\build_apk.ps1
```

Результат сохраняется в `dist\apk`.

Первая сборка занимает больше времени: Flet загружает необходимые компоненты
Flutter и Android. Собранный APK является отладочным демонстрационным пакетом.

## Важное ограничение

Это прототип интерфейса, а не банковская система. В нём нет настоящих платежей,
биометрии, МСИ, SMS, подключения к депозитарию или отправки заявок на биржу.
